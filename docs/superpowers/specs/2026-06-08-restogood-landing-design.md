# RestoGood — лендинг с ROI-калькулятором и генератором КП

**Дата:** 2026-06-08  
**Статус:** утверждён  
**Проект:** новый, отдельный от Kavi

---

## Цель

Лендинг продукта RestoGood — система онлайн-заказов для ресторанов под ключ. Главная задача страницы: конвертировать ресторан в лид через демонстрацию ROI и генерацию персонального КП.

Аудитория: владельцы и управляющие МСБ-ресторанов, которые платят комиссию агрегаторам (Яндекс.Еда, Delivery Club и др.).

---

## Секции лендинга

### 1. Navbar

Фиксированный сверху при скролле. Содержимое:
- Логотип RestoGood слева
- Якорные ссылки по центру: О продукте, Пакеты, Калькулятор
- Кнопка «Получить КП» справа (скролл к калькулятору)

На мобильном — бургер-меню.

---

### 2. Hero

Полноэкранная секция. Визуальный мокап — не скриншот реального проекта, а полностью нарисованный UI-компонент на React/Tailwind, симулирующий интерфейс системы заказов: панель с карточками блюд, статусами заказов, суммами. Рендерится прямо в браузере, выглядит как живой продукт.

**Контент:**
- Заголовок: «Ваши клиенты заказывают напрямую. Без комиссии агрегатора.»
- Подзаголовок: «Система онлайн-заказов для ресторана под ключ — от меню до трекинга статуса»
- CTA: «Рассчитать окупаемость» (якорь на калькулятор)
- Мокап справа: анимированный UI-компонент (карточки блюд, корзина, статус заказа)

---

### 3. Проблема

Три блока с иконками — боли ресторана:

1. **Агрегаторы забирают 15–35%** — с каждого заказа, каждый месяц, навсегда
2. **Звонки и ошибки** — менеджер принимает заказы вручную, путается, клиент недоволен
3. **Нет своих данных** — история заказов, любимые блюда, контакты клиентов — всё у агрегатора, не у тебя

---

### 4. Решение (О продукте)

Заголовок: «RestoGood — ваш собственный канал заказов»

Шесть фич-карточек с иконками:
- Онлайн-меню с фото и категориями
- Корзина и оформление заказа без регистрации
- Трекинг статуса по трек-коду
- Админ-панель: заказы, меню, статусы
- Роли: менеджер / владелец
- Уведомления о новых заказах

Мокап слева — нарисованный React-компонент: экран админки с таблицей заказов, статусами, суммами.

---

### 5. Как это работает

Горизонтальный степпер на десктопе, вертикальный на мобильном. Четыре шага:

1. **Подключаемся** — обсуждаем задачи, подписываем договор
2. **Настраиваем меню** — загружаем блюда, фото, цены, категории
3. **Запускаем** — ресторан получает ссылку на свой сайт заказов
4. **Принимаем заказы** — менеджер видит всё в одном окне, клиент отслеживает статус

### 7. ROI-калькулятор + КП

Центральная секция. Разделена на две колонки: форма слева, результат справа.

**Форма (левая колонка):**
- Название заведения (text input)
- Ежемесячный оборот через агрегаторы, ₽ (слайдер + поле, диапазон 50k–5M, дефолт 300k)
- Средняя комиссия агрегатора, % (слайдер, диапазон 10–35%, дефолт 25%)
- Кол-во телефонных заказов в месяц (число, опционально, дефолт 0)
- Имя и телефон/email (опционально, для КП)

**Результат (правая колонка), обновляется live:**
- Большая цифра: «Экономия в месяц: X ₽»
- Экономия в год: Y ₽
- Сравнение: «Сейчас отдаёте агрегатору: Y ₽/год → с RestoGood: 0 ₽»
- Карточки пакетов с окупаемостью: «Эконом — окупится за N мес.»

**Кнопка «Скачать КП»** — генерирует PDF на клиенте.

**Формулы:**
```
экономия_месяц = оборот × (комиссия / 100)
экономия_персонал = кол_звонков × 3мин × (500 / 60)  // 500₽/час оператора
итого_месяц = экономия_месяц + экономия_персонал
экономия_год = итого_месяц × 12
окупаемость(пакет) = цена_пакета / итого_месяц  // в месяцах, округление вверх
```

## SEO

Реализуется средствами Next.js App Router — никаких сторонних библиотек.

**Мета-теги (`app/layout.tsx` + `app/page.tsx`):**
- `<title>` и `<meta name="description">` — через Next.js Metadata API
- Open Graph теги (`og:title`, `og:description`, `og:image`) — для красивого превью при шаринге
- `<link rel="canonical">` — указывает на основной домен

**Структурированные данные:**
- JSON-LD схема `LocalBusiness` / `SoftwareApplication` в `<head>` — помогает Google понять что это за продукт

**Технические требования:**
- Семантическая разметка: один `<h1>` на страницу (в Hero), секции через `<section>`, заголовки иерархично `h2` → `h3`
- Все изображения и мокап-компоненты с атрибутом `alt`
- `sitemap.xml` — генерируется автоматически через `app/sitemap.ts`
- `robots.txt` — через `app/robots.ts`, разрешает индексацию

**Производительность (Core Web Vitals):**
- Шрифты через `next/font` (нет layout shift)
- Framer Motion анимации только после гидратации (`LazyMotion` + `domAnimation`)
- `@react-pdf/renderer` загружается динамически (`next/dynamic`, `ssr: false`) — не блокирует первый рендер
