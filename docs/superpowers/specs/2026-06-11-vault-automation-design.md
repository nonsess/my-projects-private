# Дизайн: автономный vault

**Дата:** 2026-06-11  
**Статус:** утверждён  
**Тип:** рефакторинг C — чистка + новые правила автоматизации

---

## Цель

Vault работает полностью автономно: агент сам знает что куда сохранять, всегда имеет контекст всех активных проектов, коммитит без подтверждения в конце каждой сессии. Пользователь не объясняет агенту что делать.

## Часть 1 — Чистка структуры

### Проблемы которые исправляем

| Файл / место | Проблема | Решение |
|---|---|---|
| `profile/about.md` + `profile/USER.md` | Два файла с пересекающимся содержимым | Объединить в `profile/about.md`, удалить `USER.md` |
| `vault/index.md` | Ведётся вручную, рассинхронизируется | Убрать ручной список Notes, оставить только структурные секции как якоря |
| `insights/log.md` | Не читается при старте сессии | Добавить в протокол старта |
| `project/restogood-landing.md` | Есть файл, нет в `project/index.md` | Актуализировать индекс |
| `tasks/in_progress.md` | Содержит шаблон вместо реальных задач | Очистить, привести в актуальный вид |
| `tasks/backlog.md` | Пустой | Привести в актуальный вид |
| `goals/today.md` | Не обновлялся с 2026-06-05 | Актуализировать |

### Структура остаётся той же

```
vault/
  log.md               ← append-only, трогать только append
  index.md             ← структурный каталог (не ручной список заметок)
  profile/
    about.md           ← единственный файл профиля
  project/
    index.md
    <slug>.md
  goals/
    today.md / week.md / month.md / quarter.md / year.md
  tasks/
    backlog.md / in_progress.md / blocked.md
  decisions/
    YYYY-MM-DD_slug.md
  people/
    index.md / <slug>.md
  notes/
    <slug>.md
  insights/
    log.md
```

## Часть 2 — Правила автоматизации

### Авто-сохранение без вопросов

| Триггер | Действие агента |
|---|---|
| Упомянут проект / задача | Обновить `tasks/in_progress.md` или `backlog.md` |
| Принято решение («буду делать X», «отказываюсь от Y») | Создать `decisions/YYYY-MM-DD_slug.md` |
| Всплыл факт о технологии / проблеме / инструменте | Создать или обновить `notes/<slug>.md` |
| Упомянута встреча / человек | Обновить `people/<slug>.md` |
| Изменился статус проекта | Обновить `project/<slug>.md` |
| Конец сессии | `log.md` + `goals/today.md` + коммит + пуш |

### Требует «ок» — только три вещи

- `profile/about.md` — факты о владельце
- Поля `north_star`, `stage`, `bottleneck` в карточке проекта
- `people/*.md` — любые записи о третьих лицах

### Протокол старта сессии

Читать параллельно:
1. `vault/profile/about.md`
2. `vault/project/index.md` + **все карточки** из секции `## Активные`
3. Последние 7 записей `vault/log.md`
4. `vault/goals/today.md`
5. Последние 10 записей `vault/insights/log.md`

## Часть 3 — Конец сессии и авто-коммит

### Триггеры конца сессии (любой из)

- Явное завершение: «всё», «пока», «стоп», «на сегодня»
- Разговор естественно завершается
- Длинная пауза после блока завершённой работы

### Последовательность (автоматически, без вопросов)

1. Дописать запись в `vault/log.md` — дата, о чём, настроение одной строкой
2. Обновить `vault/goals/today.md` — что сделано, что осталось
3. `git add -A`
4. `git commit -m "сессия YYYY-MM-DD: <тип>"` — тип определяется по содержанию (планирование / код / решение / идея / лог)
5. `git push 2>/dev/null || true`

### Единственное исключение

Если в сессии появились файлы с секретами (ключи, токены, пароли) — остановиться, предупредить, не коммитить.
