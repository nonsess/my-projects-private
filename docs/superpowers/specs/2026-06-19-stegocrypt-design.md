# StegoCrypt — Design Spec
_2026-06-19_

## Суть

Мобильное приложение (Flutter, Android + Web) для двух людей. Позволяет вшить зашифрованный текст в обычную фотографию и отправить её через любой мессенджер. Получатель открывает фото в приложении и читает сообщение. Внешне — просто картинка.

## Платформы

- **Android** — основная платформа, полный функционал
- **Web** — вторичная, крипто и стего работают полностью; QR-сканирование заменено вставкой текста; share target недоступен (файл открывается через кнопку выбора)

## Экраны

### Нижняя навигация
Три вкладки: «Написать» / «Прочитать» / «Ключи».

### Экран «Ключи»
- Кнопка «Сгенерировать мою пару ключей» — появляется только если ключей ещё нет
- Твой публичный ключ: QR-код + кнопка «Скопировать»
- Fingerprint твоего ключа (первые 8 байт SHA-256 публичного ключа, в hex)
- Публичный ключ друга: кнопка «Сканировать QR» (только Android) + кнопка «Вставить текст»
- Fingerprint ключа друга (для верификации по второму каналу)
- Кнопка «Удалить ключ друга» — требует подтверждения

### Экран «Написать»
1. Кнопка выбора фото из галереи + превью выбранного фото
2. Текстовое поле для сообщения (многострочное, без ограничений по длине в UI — ограничение диктует размер фото)
3. Индикатор вместимости: «Сообщение займёт X% фото»
4. Кнопка «Вшить и поделиться» → вызывает системный share sheet с готовым PNG-файлом
5. Если ключ друга не задан — заблокировано с подсказкой «Сначала добавь ключ друга»

### Экран «Прочитать»
1. Кнопка «Выбрать из галереи»
2. Приложение зарегистрировано как Android share target для PNG и JPEG
3. После загрузки файла — автоматическое извлечение и расшифровка
4. Результат: текст сообщения или сообщение об ошибке
5. Кнопка «Скопировать текст»

## Криптографическая схема

### Алгоритмы
- **X25519** — ключевая пара (асимметрика)
- **AES-256-GCM** — симметричное шифрование данных + аутентификация
- Пакет: `cryptography` (поддерживает Android, iOS, Web через WebCrypto)

### Поток при отправке (ECIES)
```
Текст (UTF-8 bytes)
  ↓
Генерация эфемерной X25519 ключевой пары (ephem_priv, ephem_pub)
Генерация случайного 96-бит nonce
  ↓
shared_secret = X25519(ephem_priv, recipient_pub_key)
session_key   = HKDF-SHA256(shared_secret, salt=nonce, info="stegocrypt")
  ↓
AES-256-GCM(текст, session_key, nonce) → ciphertext + 128-бит auth_tag
  ↓
Итоговый пакет:
  [version: 1 байт]
  [ephem_pub: 32 байта]   ← эфемерный публичный ключ отправителя
  [nonce: 12 байт]
  [ciphertext: N байт]
  [auth_tag: 16 байт]
  ↓
LSB-стеганография → PNG
```

### Поток при получении
```
PNG → LSB-извлечение → пакет байт
  ↓
Проверка версии (байт 0)
Парсинг: ephem_pub(32) | nonce(12) | ciphertext(N) | auth_tag(16)
  ↓
shared_secret = X25519(my_priv_key, ephem_pub)
session_key   = HKDF-SHA256(shared_secret, salt=nonce, info="stegocrypt")
  ↓
AES-256-GCM: расшифровка ciphertext с проверкой auth_tag
  (несовпадение → «Не удалось расшифровать: повреждено или не для вас»)
  ↓
UTF-8 текст
```

### Хранение ключей
- Приватный ключ и публичный ключ друга: `flutter_secure_storage` (Android Keystore на Android, WebCrypto-обёртка на Web)
- Свой публичный ключ: производный от приватного, хранится там же для удобства
- Ничего в SharedPreferences, ничего в логах

## Стеганография

### Алгоритм (LSB)
- Если исходник JPEG — конвертируется в PNG перед записью (lossy → lossless)
- 1 бит на канал (R, G, B) каждого пикселя = 3 бита на пиксель
- Первые 32 бита пакета = длина остальных данных (uint32 big-endian)
- Оставшиеся биты — сам пакет
- Пакет: `image` пакет (чистый Dart, работает на всех платформах)

### Вместимость
Фото 4000×3000 px → 36 Мбит → ~4.5 МБ полезной нагрузки.
Текст 10 000 символов → ~40 КБ зашифрованных данных. Влезает с огромным запасом.

### Передача
Отправлять **только как документ**, не как фото — иначе Telegram/WhatsApp пережмут PNG в JPEG и данные потеряются. Приложение при share sheet явно передаёт MIME type `image/png`.

```

  stego/
    embedder.dart          # embedBytes(image, bytes) → PNG Uint8List

    keys_screen.dart
    write_screen.dart
    read_screen.dart
  widgets/
    fingerprint_badge.dart # hex-отображение fingerprint
  main.dart
```

## Ограничения и известные упрощения

- Один получатель: приложение хранит ровно один ключ друга. Нет адресной книги.
- Нет подписи отправителя: получатель знает, что сообщение расшифровалось его ключом, но не знает криптографически кто его послал. Для двух человек достаточно.
- Нет perfect forward secrecy: один X25519 ключ на все сообщения. Если приватный ключ скомпрометирован — все прошлые сообщения под угрозой. Принято осознанно ради простоты.
- Web: без QR-сканера и share target. Только ручная вставка ключа и выбор файла.

## Вне скопа (не делаем)

- Бэкенд, сервер, регистрация
- Уведомления
- История сообщений
- Групповые чаты
- Самоуничтожающиеся сообщения
