# SEO 2026: Локальный бизнес, рестораны, доставка еды

_Исследование: июнь 2026. Источники: российские SEO-блоги + англоязычные профи-ресурсы._

---

## 1. Что реально двигает позиции для доставки/ресторанов в российских городах

- **Поведенческие факторы — решающие.** Яндекс и Google оба смотрят на: CTR из выдачи, время на сайте, глубину скролла, отказы, звонки и маршруты из карточки. Это важнее ссылок.
- **Семантика: от широкого к микролокальному.** «Доставка еды Москва» — верхний уровень, «доставка бизнес-ланчей Таганка» — реально конвертирующий хвост. Отдельные посадочные под районы / тип блюда / время суток.
- **Локальный трафик — 46% всего поиска** в России. Для доставки это основной канал.
- **Скорость + мобайл = обязательный минимум.** 70%+ трафика с мобильных. Медленный сайт теряет позиции автоматически.

## 2. Яндекс-специфика

### Яндекс.Бизнес / Яндекс.Карты
- Более **300 факторов ранжирования** карточки в Картах.
- Приоритет: **близость (70%) + отзывы (30%)** для локального поиска.
- Карточка должна быть полной: фото, актуальные часы, описание с гео-ключами, категории, атрибуты (доставка, оплата онлайн и т.д.).
- Отвечать на каждый отзыв — прямо влияет на ранг.
- **Накрутки выявляются ML** — в 2026 году алгоритмы фиксируют десятки параметров поведения. Не играть с этим.

### Поведенческие факторы (Яндекс > Google по весу)
- Время на странице, скролл, повторные визиты — Яндекс весит их тяжелее Google.
- Ранжирование **меняется в зависимости от времени суток** — если утром ищут «быстрый завтрак», ночью ищут другой формат, алгоритм адаптируется.

### Ссылки у Яндекса
- Ссылочный вес ниже, чем у Google. Качественные локальные ссылки (местные СМИ, каталоги города) работают лучше массы.

### YML-фид (для интернет-магазинов / каталогов блюд)
- Используется для попадания в Яндекс.Маркет и товарную выдачу. Для доставки еды: корректный YML с фото, ценами, категориями — условие появления в товарных блоках.
- Геотаргетинг в фиде обязателен для мультигородских сервисов.

### Яндекс Нейро (аналог Google AI Overviews)
- В 2026 нейроответы активны. Попасть туда = структурированный контент + авторитет источника + чёткие ответы на вопросы (кто, где, как, сколько).

## 3. Google-специфика

### Google Business Profile (GBP)
- **32% веса** в Local Pack — одна карточка GBP.
- Основная категория = #1 фактор попадания в Local Pack. Выбирать точно.
- **Посты 2+ раза в неделю** — сигнал активности.
- Кнопка «Заказать онлайн» — прямо в карточке, ссылка на форму заказа.
- Точные часы работы вошли в топ-5 факторов ранжирования в 2026.

### Отзывы в Google
- Конкурентный уровень: **50–100 отзывов, рейтинг 4.3+**.
- Google анализирует текст отзывов через AI — упоминания конкретных блюд, атмосферы, случаев определяют, по каким запросам ты появляешься.

### Structured Data для ресторанов
- Schema: `Restaurant`, `FoodEstablishment`, `Menu`, `MenuSection`, `MenuItem`.
- FAQ Schema — попадание в AI Overviews (40% локальных запросов уже триггерят их).
- `hasOfferCatalog` для онлайн-заказа.
- Помогает с голосовым поиском и featured snippets.

## 4. Контент-стратегия для "доставка еды [город]"

- **Отдельная страница под каждый город/район** с уникальным текстом (не шаблонный спин).
- В тексте: конкретика — время доставки, зоны, минимальный заказ, способы оплаты. Без воды.
- **Вопрос-ответ блок** на посадочной (FAQ Schema) — что входит в доставку, как отследить, есть ли бесплатная.
- Блог работает для «хвоста»: «что заказать на корпоратив», «лучшая пицца в [район]», «доставка еды за 30 минут».
- Упоминание конкретных блюд в контенте = попадание под запросы по блюдам, а не только по типу сервиса.

## 5. Техническое SEO: что важно vs что устарело

### Важно в 2026
- **LCP (Largest Contentful Paint)** — загрузка главного контента < 2.5 сек.
- **INP (Interaction to Next Paint)** — отклик на клик < 200 мс. Заменил FID с 2024.
- **CLS (Cumulative Layout Shift)** — визуальная стабильность < 0.1.
- Core Web Vitals — **тайбрейкер**, не главный фактор. Но 52% мобильного веба их не проходит — это возможность.
- Mobile-first индексация — стандарт. Версия для мобильных = основная.
- HTTPS, canonical, отсутствие дублей — базовая гигиена.

### Устарело / переоценено
- Плотность ключевых слов (keyword density) — не считается.
- Мета-keywords — игнорируется.
- Количество ссылок без качества — штраф, не плюс.
- PageRank как прямой сигнал — заменён комплексными моделями.

## 6. AI-контент и E-E-A-T в 2026

- **AI-контент не запрещён**, но Google штрафует «AI-шлак» — контент без оригинальных инсайтов, без реального опыта автора.
- **Март 2026 Core Update** усилил: оригинальность, авторский опыт, верифицируемые данные.
- **E-E-A-T для доставки:** имя автора материала + биография + ссылки на упоминания в СМИ / местных ресурсах.
- Работает: AI-текст + редактура живого эксперта + конкретные данные из опыта компании.
- Для попадания в AI Overviews: структурированные, чёткие ответы на конкретные вопросы. Заголовки в формате вопроса, прямые ответы в первом абзаце.

## 7. Алгоритмические обновления

| Обновление | Суть |
|---|---|
| Google Helpful Content (2022–2024, интегрировано) | Штрафует контент «для роботов». Человекоориентированность = обязательна. |
| Google March 2026 Core Update | Самое волатильное за долгое время. Усилен E-E-A-T, оригинальность, доверие. |
| Яндекс (2025–2026) | Усиление борьбы с накрутками поведенческих. ML-детектирование. Активное развитие нейроответов. |
| Google AI Overviews | 40%+ локальных запросов. Структурированный контент с прямыми ответами попадает туда. |

## Приоритеты для проекта (доставка / рестораны в РФ)

1. Яндекс.Бизнес: полная карточка + работа с отзывами каждую неделю.
2. Отдельные SEO-страницы под города/районы с конкретным локальным контентом.
3. Structured data: Restaurant + Menu + FAQ на каждой посадочной.
4. Core Web Vitals: пройти LCP/INP/CLS на мобильном.
5. GBP: категория точная + посты 2/нед + кнопка заказа.
6. Контент: не AI-шлак, конкретика (время, зоны, блюда), FAQ-блок.
7. Цитируемость: упоминания в местных СМИ, каталогах города — для попадания в нейроответы.
