# Дизайн лендингов — паттерны и принципы

Источник: систематизация по современным практикам SaaS 2024–2026 (Linear, Vercel, Raycast, Stripe, Loom, Notion).

---

## 1. Структура страницы — AIDA

Каждый экран должен двигать пользователя по воронке:

```
Awareness (герой)     → что это?
Interest (проблема)   → это про меня?
Desire (демо/польза)  → я хочу это
Action (цены/CTA)     → беру
```

Частые ошибки: демо до объяснения зачем, цены до демо, нет связи между блоками.

---

## 2. Герой — самый важный блок

**Правило 5 секунд:** пользователь должен понять что это и зачем за ≤5 секунд.

Три рабочих паттерна:

| Паттерн | Когда | Пример |
|---|---|---|
| Центрированный (headline + visual ниже) | продукт = главный герой | Linear, Raycast, Vercel |
| Сплит (текст слева, скрин справа) | нужна конкретность сразу | Stripe, Intercom |
| Полноэкранный (fullbleed visual) | визуальный продукт | Figma, Framer |

**Для B2C / мобильных продуктов:** центрированный работает лучше — headline бьёт сразу в обе колонки.

**Обязательно выше сгиба:**
- Заголовок (что это)
- Подзаголовок (как работает)
- Кнопка CTA (главное действие)
- Сигнал доверия (цена, условие, гарантия)

**Необязательно, но сильно:** визуал продукта (скрин, анимация, демо) — показывает что покупаешь.

## 3. Типографика

**Один шрифт, несколько весов** — это выглядит профессиональнее чем два разных шрифта.

Шкала размеров:
```
Display: 56–80px, weight 800–900, line-height 1.05, letter-spacing −0.02em
H1:      40–56px, weight 700–800, line-height 1.1,  letter-spacing −0.02em
H2:      28–36px, weight 600–700, line-height 1.2
H3:      20–24px, weight 600,     line-height 1.3
Body:    15–17px, weight 400–500, line-height 1.6–1.7
Caption: 12–13px, weight 400,     line-height 1.5, uppercase для меток
```

**Гротеск-шрифты с кириллицей (актуальные 2024–2026):**
- **Onest** — нео-гротеск, вес 100–900, очень чистый, популярен у рус. стартапов
- **Golos Text** — разработан для русского веба, нейтральный, без характера
- **Manrope** — геометрический, дружелюбный, слегка закруглённый
- **Montserrat** — широко известный, немного устарел
- **Unbounded** — очень жирный геометрический, хорош для дисплея, не для тела

**Для данных/цифр** — `ui-monospace` (системный) или `Space Mono`. Числа в моноширинном шрифте выравниваются и читаются как данные, а не текст.

## 4. Цвет и контраст

**Минимальные требования читаемости:**
```
Основной текст на белом:  >7:1  (#111827 → ~16:1 ✓)
Вторичный текст на белом: >4.5:1 (#4B5563 → ~5.9:1 ✓, #6B7280 → ~3.9:1 — пограничный)
Кнопки (цветной фон):     >3:1  для большого текста
```

**Стратегия акцента:**
- Один акцентный цвет — для CTA кнопок и ключевых данных
- Не использовать акцент для декора, только для действий и важных чисел
- Красный (#EF4444) = снижение цены, алерт, действие — семантически правильно для трекера цен

**Светлая тема vs тёмная:**
- Светлая: лучше читается, особенно мелкий текст, выше конверсия у широкой аудитории
- Тёмная: выглядит премиально, хороша для tech/dev продуктов, хуже читается мелкий текст

---

## 5. CTA — призыв к действию

**Копи кнопки:**
- Глагол + польза: "Начать бесплатно", "Попробовать →"
- Не: "Подробнее", "Узнать больше", "Войти" (для главного CTA)

**Расположение:**
- Главный CTA всегда выше сгиба
- Повторить CTA в финальной секции страницы
- Не больше двух CTA одновременно (первичный + вторичный)

**Снятие возражений под кнопкой:**
```
Без банковской карты · первый запрос бесплатно навсегда
```
Снижает порог принятия решения в 1.5–2× (по данным CXL Institute).

---

## 6. Социальное доказательство

Форматы по эффективности:
1. **Конкретные цифры** — "850+ квартир под наблюдением"
2. **Логотипы клиентов** — работает только если бренды известны аудитории
3. **Отзывы** — нужны фото/имя/должность, иначе не верят
4. **Показатели** — "снизилась на 14%" > "значительно снизилась"

Размещение: сразу после CTA в герое, или в отдельном блоке после демо.

---

## 7. Демо / визуал продукта

**Принцип:** показывай момент ценности, не скриншот интерфейса.

Для трекера цен момент ценности = email-уведомление "цена снизилась на 14%". Это то, за чем приходит пользователь. Именно это нужно показывать, а не дашборд.

Паттерны показа продукта:
- **Анимированный тикер** — показывает "живость" данных
- **Уведомление/карточка** — показывает конкретный результат
- **Интерактивный демо** — пользователь может попробовать сам (высокая конверсия)
- **Видео** — дорого, но конвертирует при правильном содержании

---

## 8. Цены — паттерны

**3-уровневая структура** (free / mid / premium) — стандарт для SaaS:
- Free: снижает барьер входа, строит базу
- Mid (featured): основной источник дохода, нужно выделить визуально
- Premium: якорная цена, делает mid дешевле на вид

**Выделение featured тарифа:**
- Акцентная рамка или верхняя полоска цвета бренда
- Бейдж "Популярный" / "Рекомендуем"
- Не поднимать выше других (нарушает сетку на мобиле)

**Снятие возражений рядом с ценой:**
"отмена в любой момент" — уменьшает восприятие риска

---

## 9. Микро-анимации

**Принцип:** анимация служит UX, а не украшает.

Рабочие паттерны:
- **Hover tilt (3D)** на карточках — тактильность, ощущение физики
- **Scroll-reveal (fade-up)** — нарративная пауза, не перегружает первый экран
- **Scan-line в живых данных** — сигнализирует об активности/обновлении
- **Float** на главном визуальном элементе — привлекает взгляд, ощущение жизни
- **Smooth accordion** в FAQ — стандарт, без него выглядит устаревшим

Что НЕ работает:
- Анимации при загрузке страницы (задерживают воспринимаемую скорость)
- Слишком много независимых анимаций одновременно
- Параллакс фонов (раздражает, вызывает motion sickness)
- Радиальные glow-эффекты (ассоциируются с шаблонными темами)

---

## 10. Мобильная адаптация

Минимум:
- Body text ≥ 15px на мобиле
- Кнопки ≥ 44px высота (touch target)
- Заголовок: ≥ 28–32px на мобиле
- Колонки → одна колонка на мобиле
- Видимый CTA выше сгиба (375px viewport)

---

## Чеклист перед публикацией лендинга

- [ ] За 5 секунд понятно что это и для кого
- [ ] CTA виден выше сгиба на десктопе И мобиле
- [ ] Все шрифты рендерятся с правильным весом
- [ ] Контраст текста ≥ 4.5:1 (проверить в DevTools → Accessibility)
- [ ] Анимации уважают `prefers-reduced-motion`
- [ ] Нет более 2 CTA одновременно
- [ ] Ценообразование находится после демо продукта
- [ ] FAQ снимает главные возражения
- [ ] Есть сигнал "без рисков" рядом с основным CTA
