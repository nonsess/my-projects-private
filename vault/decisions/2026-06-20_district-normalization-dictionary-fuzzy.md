# Флажок — нормализация районов: справочник + fuzzy, не эмбеддинги

## Решение
Районы МСК/СПб нормализуем через **канонический справочник** (146 районов Москвы
вкл. Новую Москву + 18 СПб) + транслит/ё-нормализацию в slug. Для опечаток —
**fuzzy-фолбэк** (rapidfuzz, Левенштейн/триграммы), а не эмбеддинги.

## Почему
- Закрытый список ~164 значений → словарь даёт ~100%, детерминированно, без инфры.
- Эмбеддинги недетерминированы, требуют модели, и хуже для опечаток на известных
  словах. Опечатка («савиловский») — это про расстояние, а не про смысл → fuzzy.
- Смысловые синонимы («Сити»→Пресненский) закрываются таблицей алиасов.

## Реализация
- Источник истины: `scripts/gen_districts.py` → `districts.json` (копии в backend и parser).
- Парсер: словарь exact → fuzzy (порог 86) → None. Прогоняет все гео-метки выдачи,
  ловит и «р-н …», и поселения Новой Москвы.
- Бэкенд: `app/districts.py` (детерминированно), матчинг сравнивает по slug.
- Форма: мультиселект из `/api/districts`, сгруппирован по округам, fuzzy-поиск.
