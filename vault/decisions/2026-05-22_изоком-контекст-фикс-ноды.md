# Фикс передачи истории диалога в нодах агента

Решение по задаче #28 ([[изоком-контекст-бот]]).

## Решение

Перевести все ноды, вызывающие LLM, на чередующиеся роли `user`/`assistant` вместо передачи плоской строки истории одним `user`-сообщением.

**Затронутые файлы:**
- `apps/agent/src/agent/nodes/completeness_check.py` — `messages[-10:]`
- `apps/agent/src/agent/nodes/general_response.py` — `messages[-6:]`
- `apps/agent/src/agent/nodes/router.py` — `messages[-2:]`

## Rationale

Redis checkpointer + `add_messages` reducer работали корректно, state накапливался. Проблема была в том, как ноды передавали историю в LLM: строка без ролей не давала модели понять структуру диалога, и она переспрашивала уже данные параметры.

## Результат

Закоммичено `359a07b`, запушено в `brain-boost-academy/new-izokom`. Проверено на живом Docker-стеке через WebSocket. Задача #28 переведена в `done`.
