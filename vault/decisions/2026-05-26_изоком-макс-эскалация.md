# Изоком — эскалация в МАКС параллельно с Telegram

## Решение

Добавить канал уведомлений в мессенджер МАКС параллельно с Telegram. Логика: `asyncio.gather` — оба канала запускаются одновременно, `delivered = any(results)` — достаточно одного успешного.

Если токен/chat_id канала не задан — функция тихо возвращает `False`, второй канал продолжает работать.

## Что сделано (коммит `c6c49d7`)

- `settings.py`: добавлен `MaxSettings` (читает `MAX_TOKEN`, `ESCALATION_MAX_CHAT_ID`)
- `notify.py`: `send_to_manager` → `send_to_manager_tg`; добавлен `send_to_manager_max` (POST `platform-api.max.ru/v1/messages`, заголовок `Authorization: token`)
- `notify_manager.py`: `asyncio.gather(send_to_manager_tg, send_to_manager_max)`, `delivered = any(results)`
- `.env.example`: добавлены `ESCALATION_TG_CHAT_ID`, `MAX_TOKEN`, `ESCALATION_MAX_CHAT_ID`

## Контекст

- Изоком принадлежит клиенту с российским юрлицом → MAX доступен
- Сервер не в РФ: технически не блокирует outbound-запросы к `platform-api.max.ru`
- Требования MAX Webhook (для входящих): HTTPS 443, сертификат от CA. Для исходящей эскалации webhook не нужен.
