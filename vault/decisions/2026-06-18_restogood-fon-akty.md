# Решение: фон секций — «акты по смыслу» вместо строгого чередования D/L

## Контекст

Лендинг был построен на строгом чередовании тёмный/светлый (D/L) на каждой
секции — 11 секций, ~10 переходов макс-контраста (#111110 ↔ #F6F3EE). Раньше это
фиксировалось как намеренное (см. [[project-restogood-design]]). Но Даниил сказал
прямо: от постоянной смены фона «кружится голова». Фидбэк владельца перебивает
прежнее решение.

## Решение

Сгруппировать секции в **5 смысловых актов**, каждый акт — один фон. Переходов
стало **4 вместо 10** — глаз отдыхает, нарратив читается блоками.

| Акт | Секции | Фон |
|-----|--------|-----|
| 1 — хук | Hero | тёмный |
| 2 — боль | Problem | светлый |
| 3 — наш продукт | Solution / HowItWorks / WhyUs | тёмный |
| 4 — пруф + цифры | Testimonials / LiveStats / RoiCalculator | светлый |
| 5 — цена + финал | Packages / FAQ / Footer | тёмный |

Изменённые секции: **HowItWorks** light→dark (тон `surface` #1C1B19 — лёгкий сдвиг
внутри акта 3), **LiveStats** dark→light (белые ячейки на cream, зелёный дот →
emerald-500), **FAQ** light→dark (совпал с tail-обёрткой).

Внутри акта — лёгкая тоновая вариация (dark↔surface; white-ячейки на cream), карты
не слипаются за счёт тени+outline у `.stack-card`.

## Pin на стыках актов

По просьбе Даниила: эффект pin (sticky «перелистывание», только desktop) теперь
срабатывает на **границах актов** — последняя карта акта пинится, первая карта
следующего наезжает поверх. Pin стоит на cards 0,1,4,7 (Hero, Problem, WhyUs,
RoiCalculator). Все они ≤ высоты вьюпорта (замерено: 900/910/798/941 при vh=900),
поэтому контент не прячется под наезжающей картой. Так смена фона и pin совпадают —
один акт «закрывается», следующий открывается.
