# Флажок — отмена подписки сохраняет доступ до конца периода

## Решение
Отмена подписки = «не продлевать», а не мгновенный отзыв. Доступ работает до
конца оплаченного периода (`expires_at`), затем сам падает в free.

## Почему
Был анти-UX: диалог обещал «действует до конца периода», а по факту доступ
забирали сразу (plan→free, expires_at→null). Пользователь платит за период —
должен им пользоваться.

## Реализация
- `/subscription/cancel`: меняем только `status="cancelled"`, `plan`+`expires_at` не трогаем.
- `effective_plan(sub)` (центральный хелпер) сам возвращает free после `expires_at`.
- Лимиты, история цен, форма — все гейты теперь по `effective_plan`, не по сырому `plan`.
- Воркер алертов: убран фильтр `status=="active"`, иначе отменённые-но-активные
  перестали бы получать письма.
- UI: «Отменить продление», статус «Отменена · активна до DD.MM», кнопка «Возобновить».

## Оговорка
Платежи пока разовые (30 дней), автопродления нет — «отмена» по сути косметика
до внедрения рекуррента. Но семантика доступа теперь честная.
