# Решение: удалить PDF-файлы из git-истории через force push

Задача #35 [[изоком-url-документов-фикс]].

## Контекст

PDF-документы были закоммичены в репо `brain-boost-academy/new-izokom` (коммит `805278d`) с целью запечь их в Docker-образ через `COPY data/documents`. После фикса URL-бага стало ясно, что фича «документы доступны через веб» не нужна прямо сейчас — и файлы лишь утяжеляют репо.

## Решение

Откат через `git reset --hard f8d06ca` + `git push --force origin main`. Это удаляет коммит с PDF из истории (GitHub garbage-collect'ит его со временем), а не просто добавляет revert-коммит поверх.

Причина выбора force push вместо revert: `git revert` сохраняет файлы в истории — репо не становится легче. Force push убирает коммит полностью.

## Итог

- `.gitignore` возвращён к `data/` без исключений.
- `Dockerfile.prod` без `COPY data/documents`.
- `compose.yaml` без volume mount для документов.
- Пуш `f8d06ca` стал последним в `main`.
